Introducing AskTIM: MIT's AI Learning Assistant

Empowering All Learner Success with AI-Powered Support

As part of Universal AI's commitment to enhancing the learning experience, we're excited to introduce AskTIM—an intelligent, always-available AI learning assistant designed to support learners throughout their academic journey.

AskTIM is seamlessly integrated into Universal AI course modules, offering real-time, contextual help that empowers learners to take ownership of their learning.

What is AskTIM?

AskTIM is your personal AI learning assistant, built to help you succeed. Whether you're watching a video, reviewing self-assessment questions, or working on an assignment, AskTIM is there to support you.

You can use AskTIM to:

  • Get quick explanations when you're stuck
  • Review key ideas from lectures
  • Understand tricky questions without giving away the answers

Think of AskTIM as a helpful study buddy—always ready to guide you, but never doing the thinking for you.

How AskTIM Supports Learners

Interactive Q&A

  • Learners can ask questions about video lectures, self-assessment questions, or assignments. AskTIM responds with clear, concise explanations tailored to the course content.

Video Summaries

  • After watching a lecture, learners can request a quick summary to reinforce key takeaways or catch up on missed content.

AI-Generated Flashcards

  • AskTIM creates custom flashcards based on module content, helping learners review and retain information more effectively.

Assessment Guidance

  • When learners are stuck on a knowledge check or assessment, AskTIM offers hints and reasoning strategies—without giving away the answer—encouraging deeper understanding.

Conversation Memory

  • AskTIM remembers previous interactions, allowing learners to build on past questions and revisit explanations as needed.

How It Works

Learners will see the AskTIM button below each video, self-assessment and assignment question. With one click, they can start a conversation and receive personalized support—anytime, anywhere.

Getting Started

Good news—AskTIM is already built into your Universal AI modules, so there's nothing you need to install or set up.

Here's how to make the most of it:

  • Look for the AskTIM button below videos and assessment questions.
  • Click it anytime you need help understanding a concept, reviewing material, or figuring out a tricky question.
  • Use it as your go-to resource for quick explanations, summaries, and study tools like flashcards.

AskTIM is here to support your learning—whenever you need it, wherever you are.

Introduction to Jupyter Notebook

Welcome to Jupyter Notebook!

Jupyter is an open-source interactive computing environment that lets you write and run code, create visualizations, and include text, equations, and multimedia — all in one place.

What is a Jupyter Notebook?

  • A notebook is made up of cells that can contain code, text (Markdown), equations (LaTeX), and outputs (figures, tables, logs).
  • Jupyter is widely used for data analysis, scientific research, teaching, and prototyping.
  • It supports many programming languages, but Python is the most common.

Why use Jupyter?

  • Interactive data exploration and visualization
  • Rapid prototyping and experimentation
  • Combine code, explanations, and results for reproducible research
  • Runs locally or on remote servers (e.g., JupyterHub, VS Code)

Basic Components

  • Code cells → run Python (or other supported languages, like Julia).
  • Markdown cells → format text, add headings, bullet lists, images, and equations.
  • Output → immediately visible under the code cell (print statements, plots, tables).

Shortcuts

  • Shift + Enter → run a cell
  • Esc + A → insert a cell above
  • Esc + B → insert a cell below

Example Code Cell

x = 10
y = 5
print("x + y =", x + y)

Example Markdown Cell

# This is a heading
 You can write *italic*, **bold**, and even add equations:

$$E = mc^2$$

You can double-click to edit markdown code, and run the cell to compile it like a code cell.

Order of Running Cells

In Jupyter, the order in which you run cells REALLY MATTERS.

  • Each time you run a code cell, its variables, functions, and imports are stored in memory.
  • If you run cells out of order, you might use variables or functions that haven't been defined yet, causing errors.
  • Running a cell multiple times can overwrite variables with new values, which may lead to unexpected results.
  • The execution number (shown to the left of each cell, like In [3]:) tells you the order in which the cells were run.

Best practice:

  • Run cells top to bottom to keep your code consistent.
  • If things get confusing, use Kernel → Restart & Run All to reset the notebook and run everything in order.
Example

Imagine we have two cells:

# Cell 1
x = 10

# Cell 2
print(x * 2)

If you run Cell 1 first and then Cell 2, the output will be 20.

But if you run Cell 2 before Cell 1, you'll get an error: NameError: name 'x' is not defined.

If you later change Cell 1 to x = 5 and run only Cell 2 again, the output will be updated to 10.

Downloading and exporting

  1. You can download the notebook by going to "File" in the top left corner and clicking on "Download". This will save the notebook locally to your laptop.
  2. You can export the notebook as a PDF document to going to "File" → "Save And Export Notebok As" → "PDF". This will save the notebook in a PDF format to your laptop. Note that the PDF document is static, meaning you are not able to code in it. It is, however, a nice way of showing someone your work.

Saving the notebook

You can save a notebook by going ot the top left corner, and selecting "Save Notebook", or with the keyboard shortcuts: Command + S (Mac) or Ctrl+S (Windows).

IMPORTANT NOTE: When you open a notebook from Universal AI, you will not be able to save it. This is because the notebook is being remotely run on the Universal AI servers. If you would like to make changes and save those changes, you will need to download the notebook, make your changes in a code editor like JupyterHub or VS Code, and then save.

Kernels and restarting kernels

A kernel is the computational engine that executes the code in your notebook. In most cases, this is the Python interpreter running in the background.

  • When you run a code cell, the kernel executes the code and returns the output below the cell.
  • The kernel maintains the state of your session, meaning variables and imports stay in memory until you restart.

Restarting the Kernel

  • If your notebook gets stuck or you want a clean slate, you can restart the kernel.
  • Restarting clears all variables, functions, and imports from memory.
  • After restarting, you'll need to re-run the cells in order to redefine variables and reload libraries.

In Jupyter, use the menu: Kernel → Restart (or in JupyterLab: Kernel → Restart Kernel...)
